## Quality was enforced, not hoped for

<div class="grid grid-cols-2 gap-4 mt-5 text-sm">
  <div class="card">
    <b>Test pyramid centred on integration</b>
    <div class="op75 text-xs mt-1.5 leading-relaxed">
      Full request flow against <b>real PostgreSQL + NATS</b>; only the two
      non-repeatable parts stubbed — the model and the browser. Dedicated test DB,
      migrations applied at session start (proving them idempotent), per-test truncation.
    </div>
  </div>
  <div class="card">
    <b>Boundaries that fail the build</b>
    <div class="op75 text-xs mt-1.5 leading-relaxed">
      Four <span class="font-mono">import-linter</span> contracts reject any cross-package
      import — e.g. <span class="font-mono">core-stays-pure</span>: the shared library can
      never pick up model or browser code.
    </div>
  </div>
  <div class="card">
    <b>Schema as code</b>
    <div class="op75 text-xs mt-1.5 leading-relaxed">
      The whole database declared in Atlas HCL; versioned SQL migrations generated by diff
      and applied by a dedicated migrator image — a Helm pre-install hook gating every rollout.
    </div>
  </div>
  <div class="card">
    <b>Three engineering pivots, owned</b>
    <div class="op75 text-xs mt-1.5 leading-relaxed">
      SQLite → PostgreSQL when the worker tier went horizontal · opaque recorded recipes →
      schema-validated DSMs · migration ordering solved with the migrator-hook pattern.
    </div>
  </div>
</div>
